





这篇文章的视角非常有意思。在使用Claude Code时快速切换Api服务商看似无缝，其实代价非常之大。

不仅仅是切换服务商，甚至在同一会话下，切换模型代价都是巨大的。比如说，如果你已经在一个 会话中进行了 10 万 token 的对话，现在使用的是Opus 4.6模型，但是这时你想问一个相对简单的问题，直觉上你可能觉得切换到 Haiku 更便宜。但实际上，切换到 Haiku 反而更贵，因为你需要为 Haiku 重新构建整套提示缓存，需要花费10万*1.25倍的token用量来完成构建。

你可以考虑使用/btw指令悄悄新开一个会话，然后合并到主会话。好像有点类似于一个SubAgent子代理一样，最终将结果回传给主Agent一样。

最后再强调一点在Claude Code 中默认构建缓存，缓存默认生命周期5分钟，每次请求命中缓存后，缓存时间会重新刷新，重新计时5分钟，超过5分钟其实就可以看情况，就要新开会话了。
