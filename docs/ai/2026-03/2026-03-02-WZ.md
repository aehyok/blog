# VitePress + Obsidian + GitHub + Cloudflare 从零到一建站完整指南

> 本文将手把手带你搭建一套完整的个人知识库/博客系统：用 VitePress 生成静态网站，用 Obsidian 管理 Markdown 内容，通过 GitHub 托管代码，最终借助 Cloudflare Pages 实现自动化部署。

---

## 整体架构概览

```
写作流程：Obsidian 编辑 Markdown → Git 提交推送 → GitHub 仓库
部署流程：GitHub 变更触发 → Cloudflare Pages 自动构建 → 网站更新上线
```

**技术栈说明：**

- **VitePress**：基于 Vite + Vue 3 的静态站点生成器，速度快、配置简单
- **Obsidian**：本地 Markdown 编辑器，双链笔记、插件生态丰富
- **GitHub**：代码托管 + 版本控制，作为内容仓库
- **Cloudflare Pages**：免费静态网站托管，支持 GitHub 自动部署，全球 CDN 加速

## 第一步：环境准备

### 1.1 安装 Node.js

VitePress 需要 Node.js 18+ 版本。

前往 [https://nodejs.org](https://nodejs.org/) 下载 LTS 版本，安装完成后验证：

```bash
node -v    # 应显示 v18.x.x 或更高
npm -v     # 应显示 npm 版本
```

### 1.2 安装 Git

前往 [https://git-scm.com](https://git-scm.com/) 下载安装，完成后配置用户信息：

```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱@example.com"
```

### 1.3 安装 Obsidian

前往 [https://obsidian.md](https://obsidian.md/) 下载对应操作系统的安装包，安装即可。

## 第二步：创建 VitePress 项目

### 2.1 初始化项目

选择一个合适的目录，创建项目文件夹并初始化：

```bash
# 创建项目目录
mkdir my-site
cd my-site

# 初始化 npm 项目
npm init -y

# 安装 VitePress
npm install -D vitepress
```

### 2.2 使用向导初始化 VitePress

```bash
npx vitepress init
```

向导会问你几个问题，按如下参考回答：

```
┌  Welcome to VitePress!
│
◇  Where should VitePress initialize the config?
│  ./docs                          ← 文档放在 docs 目录
│
◇  Site title:
│  My Awesome Site                 ← 你的网站标题
│
◇  Site description:
│  A VitePress Site                ← 网站描述
│
◇  Theme:
│  Default Theme                   ← 使用默认主题
│
◇  Use TypeScript for config file?
│  No                              ← 选 No 更简单
│
◇  Add VitePress npm scripts to package.json?
│  Yes                             ← 一定选 Yes
```

### 2.3 了解生成的目录结构

初始化完成后，项目结构如下：

```
my-site/
├── docs/                    ← 所有内容在这里
│   ├── .vitepress/
│   │   └── config.js        ← VitePress 核心配置文件
│   ├── index.md             ← 网站首页
│   └── api-examples.md      ← 示例页面（可删除）
├── node_modules/
└── package.json
```

### 2.4 修改 package.json 脚本

打开 `package.json`，确认 scripts 部分如下（向导通常已自动添加）：

```json
{
  "scripts": {
    "docs:dev": "vitepress dev docs",
    "docs:build": "vitepress build docs",
    "docs:preview": "vitepress preview docs"
  }
}
```

### 2.5 本地预览效果

```bash
npm run docs:dev
```

打开浏览器访问 `http://localhost:5173`，能看到 VitePress 默认页面，说明安装成功。

## 第三步：配置 VitePress

### 3.1 编辑核心配置文件

打开 `docs/.vitepress/config.js`，按需修改：

```javascript
import { defineConfig } from 'vitepress'

export default defineConfig({
  // 网站基本信息
  title: "我的知识库",
  description: "用 VitePress 构建的个人笔记与博客",
  
  // 如果部署在子路径下需要配置 base，根域名部署留空或设为 '/'
  // base: '/',

  // 主题配置
  themeConfig: {
    // 网站 Logo（可选）
    logo: '/logo.png',
    
    // 顶部导航栏
    nav: [
      { text: '首页', link: '/' },
      { text: '笔记', link: '/notes/index' },
      { text: '博客', link: '/blog/index' },
    ],

    // 左侧侧边栏
    sidebar: {
      '/notes/': [
        {
          text: '我的笔记',
          items: [
            { text: '介绍', link: '/notes/index' },
            { text: '示例文章', link: '/notes/example' },
          ]
        }
      ],
      '/blog/': [
        {
          text: '博客文章',
          items: [
            { text: '文章列表', link: '/blog/index' },
          ]
        }
      ]
    },

    // 社交链接
    socialLinks: [
      { icon: 'github', link: 'https://github.com/你的用户名' }
    ],

    // 页脚
    footer: {
      message: 'Released under the MIT License.',
      copyright: 'Copyright © 2024 你的名字'
    },

    // 搜索功能（本地搜索，免费）
    search: {
      provider: 'local'
    }
  }
})
```

### 3.2 创建内容目录结构

根据导航配置，创建对应目录：

```bash
mkdir -p docs/notes
mkdir -p docs/blog
```

创建 `docs/notes/index.md`：

```markdown
---
title: 笔记总览
---

# 我的笔记

欢迎来到我的知识库！
```

创建 `docs/blog/index.md`：

```markdown
---
title: 博客
---

这里是我的博客。
```

### 3.3 配置首页（index.md）

修改 `docs/index.md`，VitePress 支持漂亮的 Hero 首页布局：

```markdown

  name: "我的知识库"

    - theme: brand
      text: 开始阅读
      link: /notes/index
    - theme: alt
      text: 关于我
      link: /about

features:
  - icon: 📝
    title: 笔记整理
    details: 系统化整理学习笔记与思考
  - icon: 🚀
    title: 快速访问
    details: 全球 CDN 加速，访问速度极快
  - icon: 🔍
    title: 全文搜索
    details: 支持本地全文搜索
```

## 第四步：用 Obsidian 管理内容

### 4.1 将 docs 目录作为 Obsidian 仓库

Obsidian 的核心概念是"仓库（Vault）"，就是一个本地文件夹。我们直接把 VitePress 的 `docs` 目录作为 Obsidian 仓库。

1. 打开 Obsidian
2. 点击左下角「打开其他仓库」
3. 选择「打开文件夹作为仓库」
4. 选择你的 `my-site/docs` 文件夹

这样 Obsidian 就直接管理 VitePress 的内容目录了。

### 4.2 配置 Obsidian 让其兼容 VitePress

**关键设置：关闭 Wiki 链接，改用标准 Markdown 链接**

VitePress 不支持 Obsidian 的 `[[双链]]` 格式，需要改为标准链接。

进入 Obsidian 设置 → 文件与链接：

- **内部链接类型**：改为「基于仓库根目录的绝对路径」
- **使用 [[Wikilinks]]**：关闭此选项

**推荐安装的 Obsidian 插件：**

进入 设置 → 社区插件 → 浏览，搜索并安装：

|插件名|用途|
|---|---|
|Obsidian Git|在 Obsidian 内直接 Git 提交推送|
|Linter|自动格式化 Markdown，保证格式规范|
|Dataview|创建动态文章列表（可选）|

### 4.3 图片管理（图床方案）

题目中提到图床方案已有，这里说明 Obsidian 如何配合使用。

**推荐流程：**

1. 截图或复制图片
2. 使用图床工具上传（如 PicGo + 阿里云 OSS / 腾讯云 COS / Cloudflare R2）
3. 图床工具自动返回并复制图片 URL
4. 在 Obsidian 中粘贴为标准 Markdown 格式：`![图片描述](https://你的图床域名/图片.png)`

**配置 PicGo（以 Cloudflare R2 为例）：**

```
PicGo 设置 → 图床设置 → Amazon S3（R2 兼容 S3 协议）
- AccessKey ID: 你的 R2 AccessKeyId
- SecretAccessKey: 你的 R2 SecretAccessKey
- Bucket: 你的 Bucket 名
- 自定义节点: https://<账户ID>.r2.cloudflarestorage.com
- 自定义域名: https://你绑定的自定义域名
```

### 4.4 Obsidian 写作规范

为了让 VitePress 正确渲染，每篇文章建议使用 Front Matter：

```markdown
---
title: 文章标题
date: 2024-01-01
description: 文章简介
tags: [标签1, 标签2]
---

# 文章标题

正文内容...
```

## 第五步：创建 GitHub 仓库

### 5.1 在 GitHub 创建新仓库

1. 登录 [https://github.com](https://github.com/)
2. 点击右上角「+」→「New repository」
3. 填写信息：
    - Repository name: `my-site`（或任意名称）
    - 选择 **Public**（Cloudflare Pages 免费版支持公开仓库，私有仓库也支持）
    - **不要**勾选「Initialize this repository with a README」
4. 点击「Create repository」

### 5.2 配置 .gitignore

在项目根目录创建 `.gitignore` 文件：

```
# 依赖目录
node_modules/

# VitePress 构建输出（Cloudflare 会自动构建，不需要上传）
docs/.vitepress/dist/
docs/.vitepress/cache/

# 系统文件
.DS_Store
Thumbs.db

# 编辑器
.vscode/
```

### 5.3 初始化 Git 并推送

在项目根目录执行：

```bash
# 初始化 Git 仓库
git init

# 添加所有文件
git add .

# 首次提交
git commit -m "feat: 初始化 VitePress 项目"

# 关联远程仓库（替换为你的 GitHub 用户名和仓库名）
git remote add origin https://github.com/你的用户名/my-site.git

# 推送到 GitHub
git branch -M main
git push -u origin main
```

推送完成后，在 GitHub 仓库页面刷新，应该能看到所有文件。

## 第六步：配置 Cloudflare Pages 自动部署

### 6.1 注册 Cloudflare 账号

前往 [https://cloudflare.com](https://cloudflare.com/) 注册账号（免费）。

### 6.2 连接 GitHub 仓库

1. 登录 Cloudflare Dashboard
2. 左侧菜单选择「**Workers & Pages**」
3. 点击「**Create application**」
4. 选择「**Pages**」标签
5. 点击「**Connect to Git**」

### 6.3 授权 GitHub

1. 点击「Connect GitHub」
2. 跳转到 GitHub 授权页面，选择「Only select repositories」
3. 选择你的 `my-site` 仓库
4. 点击「Install & Authorize」

### 6.4 配置构建设置

回到 Cloudflare，选择刚才的仓库后，填写构建配置：

|配置项|值|
|---|---|
|Project name|my-site（自定义）|
|Production branch|main|
|**Framework preset**|**VitePress**|
|Build command|`npm run docs:build`|
|Build output directory|`docs/.vitepress/dist`|

**环境变量（重要）：**

点击「Environment variables」→「Add variable」：

|变量名|值|
|---|---|
|NODE_VERSION|18|

> 必须指定 Node.js 版本，否则 Cloudflare 可能使用旧版本导致构建失败。

### 6.5 开始部署

点击「**Save and Deploy**」，Cloudflare 会：

1. 拉取 GitHub 仓库代码
2. 安装依赖（`npm install`）
3. 执行构建命令（`npm run docs:build`）
4. 将 `docs/.vitepress/dist` 部署到全球 CDN

**首次部署大约需要 1-3 分钟。**

部署完成后，Cloudflare 会分配一个免费域名：`https://my-site.pages.dev`

### 6.6 验证自动部署

测试自动部署是否生效：

1. 在本地修改任意 Markdown 文件
2. 执行 Git 提交推送：
    
    ```bash
    git add .git commit -m "test: 测试自动部署"git push
    ```
    
3. 打开 Cloudflare Dashboard → Workers & Pages → 你的项目
4. 在「Deployments」标签可以看到新的部署正在进行
5. 部署完成后访问网站，确认内容已更新

## 第七步：绑定自定义域名（可选）

### 7.1 在 Cloudflare Pages 添加自定义域名

1. 进入你的 Pages 项目
2. 点击「**Custom domains**」标签
3. 点击「**Set up a custom domain**」
4. 输入你的域名，例如：`blog.example.com`
5. 点击「Continue」

### 7.2 配置 DNS

**情况一：域名托管在 Cloudflare（推荐）**

Cloudflare 会自动添加 CNAME 记录，直接点击「Activate domain」即可。

**情况二：域名在其他服务商**

需要手动添加 DNS 记录：

```
类型: CNAME
名称: blog（或 www，或 @）
目标: my-site.pages.dev
```

等待 DNS 生效（通常 5-30 分钟），Cloudflare 会自动签发 SSL 证书。

## 第八步：优化工作流

### 8.1 使用 Obsidian Git 插件自动推送

安装 Obsidian Git 插件后，进行配置：

进入 Obsidian 设置 → Obsidian Git：

```
Vault backup interval: 30        ← 每 30 分钟自动提交
Auto pull interval: 10           ← 每 10 分钟自动拉取
Commit message: docs: auto save {{date}}
```

这样在 Obsidian 写完文章后，无需手动 Git 操作，插件会自动提交推送，Cloudflare 检测到变更后自动重新部署。

### 8.2 本地预览与写作流程

推荐的日常工作流：

```
1. 打开终端，在项目目录运行: npm run docs:dev
2. 打开 Obsidian，开始写作
3. 浏览器访问 localhost:5173 实时预览效果
4. 写作完成后，Obsidian Git 自动提交（或手动 git push）
5. Cloudflare 自动检测变更，1-2 分钟内网站更新
```

### 8.3 常用 Git 命令速查

```bash
# 查看当前状态
git status

# 添加所有变更
git add .

# 提交（写清楚提交信息）
git commit -m "docs: 新增 Python 学习笔记"

# 推送到 GitHub（触发 Cloudflare 自动部署）
git push

# 拉取最新代码（多设备协作时用）

```

## 常见问题排查

### Q1：构建失败，提示找不到命令

**现象：** Cloudflare 构建日志报错 `sh: vitepress: command not found`

**解决：** 确认 `package.json` 中的构建命令为 `npm run docs:build`，而不是直接调用 `vitepress build`。

### Q2：图片不显示

**现象：** 本地正常，部署后图片 404

**解决：** 检查图片路径。使用图床方案时确保图片 URL 是完整的 `https://` 链接；本地图片需放在 `docs/public/` 目录下，引用时使用 `/图片名.png`（无需包含 `public`）。

### Q3：中文路径出现 404

**现象：** 文件名包含中文时，部署后访问 404

**解决：** 建议所有文件名使用英文或拼音，避免中文文件名带来的编码问题。

### Q4：Obsidian 的双链在网站上不渲染

**现象：** `[[文章名]]` 在网站上显示为纯文本

**解决：** 在 Obsidian 设置中关闭 Wikilinks，改用标准 Markdown 链接 `[文章名](./文章名.md)`。

### Q5：部署后 CSS 样式丢失

**现象：** 部署到子路径时样式 404

**解决：** 在 `config.js` 中配置 `base` 选项：

```javascript
export default defineConfig({
  base: '/my-site/',  // 如果部署在 https://example.com/my-site/
})
```

根域名部署无需配置此项。
