让你的终端永不失联：一篇写给普通人的 tmux 指南 ​

读完你就明白，为什么程序员都在用它

01 它到底是什么？ ​

你有没有遇到过这种情况：在服务器上跑一个耗时好几个小时的程序，结果 WiFi 断了一下，SSH 断连，任务就这么消失了？

tmux 就是用来解决这个问题的。它的全称是 Terminal Multiplexer，直译是"终端复用器"——简单说，它能让你在一个终端窗口里同时开好几个独立的工作区，并且这些工作区在你断开连接后依然在后台持续运行。

💡 类比

想象你在公司有一张大桌子，桌上摆着好几个不同项目的文件夹。你可以随时离开去吃饭，回来时桌子上的东西还在原地。tmux 就是这张永远不会被收拾的桌子——哪怕你断网、关电脑，你的"工作现场"都原封不动地保留着。

02 三个概念，搞懂就够了 ​

tmux 有三层结构，从大到小依次是：

tmux 服务器
├── 会话 Session  ← 一个项目 / 一个任务
│   ├── 窗口 Window   ← 一个标签页
│   │   ├── 面板 Pane  ← 一块分屏区域
│   │   └── 面板 Pane  ← 另一块分屏区域
│   └── 窗口 Window
└── 会话 Session  ← 另一个项目
概念类比说明
Session 会话一个项目文件夹对应一个项目。断开连接后，会话继续在后台存活，随时可以重新连上。
Window 窗口浏览器标签页类似浏览器标签页。一个会话里可以有多个窗口，随时切换。
Pane 面板分屏区域把窗口分割成多块区域，每块都是一个独立的终端，同时运行不同程序。

03 五分钟上手，记住这些就行 ​

tmux 所有操作都靠一个前缀键触发，默认是 Ctrl + b。先按这个组合键，松开，再按其他键来执行操作。就像 Word 里的 Ctrl，是所有快捷键的"前置动作"。

启动一个有名字的会话 ​

bash
# 创建名为 "myproject" 的会话
tmux new -s myproject

# 现在你在 tmux 里了，可以正常工作
python server.py

最常用快捷键一览 ​

快捷键作用
Ctrl+b 然后 d分离会话（detach），程序继续后台运行
tmux attach -t myproject重新连回刚才的会话
tmux ls列出所有正在运行的会话
Ctrl+b 然后 %左右分屏，左右各一个面板
Ctrl+b 然后 "上下分屏
Ctrl+b 然后 方向键在面板之间切换焦点
Ctrl+b 然后 c新建一个窗口
Ctrl+b 然后 0~9切换到对应编号的窗口
Ctrl+b 然后 s可视化选择会话
Ctrl+b 然后 x关闭当前面板

04 真实场景：它能帮你做什么？ ​

🚂 场景一：长时间任务不怕断网 ​

假设你在服务器上跑一个机器学习模型训练，要跑 8 个小时。

  • 以前： SSH 断了，任务就没了，哭都没用。
  • 用 tmux 后： 在 tmux 里启动训练，然后按 Ctrl+b d 分离，关电脑回家。第二天再 SSH 上去，tmux attach 一下，8 小时的任务还在跑，结果就在那等着你。
bash
# 在 tmux 里启动训练
tmux new -s training
python train.py --epochs 100

# 断开连接（任务继续跑）
# Ctrl+b d

# 第二天重新连上
ssh user@server
tmux attach -t training
# Epoch 98/100... loss: 0.0341  ← 任务还在！

🖥️ 场景二：同时看日志 + 写代码 + 跑测试 ​

开发时你经常需要同时做好几件事。用 tmux 分屏，一个面板跑开发服务器，一个面板看日志实时输出，一个面板继续写代码。

再也不用在 N 个终端窗口之间来回切换，一个屏幕搞定所有。

bash
# 左右分屏
Ctrl+b %

# 左边面板：运行开发服务器
npm run dev

# 切换到右边面板
Ctrl+b →

# 右边面板：实时查看日志
tail -f /var/log/app.log

👥 场景三：和同事共享终端 ​

远程协作时，你和同事可以同时 attach 到同一个 tmux 会话，看到相同的终端内容。这比"截图发微信"方便得多，非常适合远程 debug 或者演示。

bash
# 同事连上服务器后，执行：
tmux attach -t demo -r   # -r 表示只读模式

🤖 场景四：配合 AI 编程助手 ​

最近流行 AI 编程助手（比如 Claude Code、Cursor 等）。有开发者专门研究发现：

让 AI 助手在 tmux 会话里启动后台服务，你可以实时看到服务器输出，也可以随时手动介入操作，比 AI 自己管理后台进程要透明可靠得多。

05 让 tmux 更好用的几个配置 ​

tmux 有一个配置文件 ~/.tmux.conf，几行配置就能让体验大幅提升：

bash
# 开启鼠标支持（可以用鼠标点击面板、拖动边框）
set -g mouse on

# 支持真彩色，让终端颜色更好看
set -g default-terminal "tmux-256color"

# 用更直观的按键分屏（| 是垂直分割，- 是水平分割）
bind | split-window -h
bind - split-window -v

# 增大历史记录行数
set -g history-limit 10000

# 消除 Esc 键延迟（对 vim 用户很重要）
set -sg escape-time 0

修改完后，在 tmux 里按 Ctrl+b 然后输入 :source ~/.tmux.conf 即可生效，不需要重启。

06 遇到问题？先看这里 ​

Q：颜色显示不对、有乱码怎么办？

在配置文件里加上 set -g default-terminal "tmux-256color"，并确保你的终端本身支持 256 色。

Q：鼠标滚动没反应？

先确认 set -g mouse on 已经写进配置文件，然后重新加载配置。如果还不行，可能是终端本身不支持，换用 iTerm2（Mac）或 Windows Terminal 试试。

Q：会话突然显示 "attached" 无法连接？

用 tmux attach -d -t 会话名 命令，-d 会先把其他客户端踢下线，然后你就能连上了。

Q：tmux 和直接用终端标签页有什么区别？

终端标签页依赖本地终端软件，断开 SSH 就丢失了。tmux 运行在服务器上，和你的连接状态无关，这是最本质的区别。本地开发的话，标签页可能够用；远程服务器上，强烈建议用 tmux。

07 从哪里开始？ ​

别试图一口气学完所有功能，先把这几步走通：

  1. 装好 tmux

    • Ubuntu/Debian：sudo apt install tmux
    • macOS：brew install tmux

  2. 用它启动你下一个需要持续运行的任务

    • tmux new -s 任务名，跑程序，然后 Ctrl+b d 分离

  3. 加上鼠标支持

    • 把 set -g mouse on 写进 ~/.tmux.conf，立刻舒服很多

  4. 尝试分屏

    • 用 Ctrl+b % 左右分屏，一边跑程序一边看日志

  5. 探索插件（可选）

    • 安装 tmux-resurrect 可以保存和恢复整个 tmux 布局，即使服务器重启也没问题

💡 tmux 的学习曲线不高，但收益是长期的。一旦养成习惯，你会发现离开它反而不习惯——就像用过双屏显示器，回到单屏会觉得"这怎么够用"。
